# MongoDB Atlas plugin for Tabularis

A fork of [TabularisDB/tabularis-mongodb-plugin](https://github.com/TabularisDB/tabularis-mongodb-plugin) with real MongoDB Atlas and full connection URI support.

The plugin is a Rust 2021 binary that uses the official `mongodb = "3"` driver and communicates with Tabularis through JSON-RPC 2.0 over stdio. It keeps the existing database/collection browsing, schema inference, CRUD, index, and query operations while adding:

- MongoDB Atlas connections
- `mongodb+srv://` discovery
- full `mongodb://` URIs, including multiple hosts
- TLS/SSL options
- `authSource`, `replicaSet`, `retryWrites`, `w`, `appName`, and `directConnection`
- legacy host/port/username/password configuration
- a separate `mongodb-atlas` plugin id, so it can be installed beside the official plugin

## Connection configuration

### Full URI (recommended for Atlas)

The first non-empty full URI field is used in this order:

1. `connection_string`
2. `connectionString`
3. `connection_uri`
4. `connectionUri`
5. `uri`
6. `url`
7. `dsn`

Only `mongodb://` and `mongodb+srv://` are accepted. A full URI is validated and passed unchanged to `Client::with_uri_str`; host/port fields and separate query options are not merged into it.

#### MongoDB Atlas SRV

```text
mongodb+srv://user:password@cluster0.xxxxx.mongodb.net/mydb?retryWrites=true&w=majority&authSource=admin
```

#### MongoDB Atlas standard URI

```text
mongodb://host1:27017,host2:27017,host3:27017/mydb?tls=true&authSource=admin&replicaSet=atlas-xxxxx-shard-0&retryWrites=true&w=majority
```

#### Local MongoDB

```text
mongodb://localhost:27017/test
```

Use the connection string generated by Atlas for the target cluster. Replace the placeholders locally and percent-encode reserved characters in credentials. A recommended Atlas shape is:

```text
mongodb+srv://<percent-encoded-user>:<percent-encoded-password>@<cluster>.mongodb.net/<database>?retryWrites=true&w=majority&authSource=admin&appName=Tabularis
```

### Host and port compatibility

Existing Tabularis configurations continue to work:

```json
{
  "host": "localhost",
  "port": 27017,
  "database": "test",
  "username": "app-user",
  "password": "local-password",
  "tls": true,
  "authSource": "admin",
  "replicaSet": "replica-set-name",
  "retryWrites": true,
  "w": "majority",
  "appName": "Tabularis"
}
```

When a URI is built from separate fields, username and password are percent-encoded. Supported top-level query options are:

- `tls`
- `ssl`
- `authSource`
- `replicaSet`
- `retryWrites`
- `w`
- `appName`
- `directConnection`

Additional MongoDB driver options can be supplied as a JSON object in `extra_options`. Object keys are sorted for deterministic URI construction, scalar values are percent-encoded, and duplicate option names are rejected:

```json
{
  "host": "localhost",
  "extra_options": {
    "compressors": "zstd",
    "readPreference": "secondaryPreferred"
  }
}
```

`extra_options` also accepts a `key=value&key=value` query string for compatibility. Prefer the JSON object form because its values are encoded safely by the plugin.

## Features

- List databases and collections.
- Infer fields and BSON types by sampling documents.
- Inspect and drop indexes.
- Execute `find`, `findOne`, `aggregate`, and count queries using MongoDB shell syntax.
- Insert, update, and delete documents.
- Convert string `_id` values to `ObjectId` when possible.
- Generate MongoDB shell commands for collection, index, and field operations.
- Reuse one MongoDB client per connection URI during the plugin process lifetime.

## Manual installation

Build the release binary, then copy the binary and `manifest.json` into the `mongodb-atlas` directory for the current platform:

| OS | Plugin directory |
|---|---|
| macOS | `~/Library/Application Support/com.debba.tabularis/plugins/mongodb-atlas/` |
| Linux | `~/.local/share/tabularis/plugins/mongodb-atlas/` |
| Windows | `%APPDATA%\com.debba.tabularis\plugins\mongodb-atlas\` |

On macOS or Linux, the installation script builds and copies both files:

```bash
./sync.sh
```

On Windows, build and install with PowerShell:

```powershell
cargo build --release
$destination = Join-Path $env:APPDATA "com.debba.tabularis\plugins\mongodb-atlas"
New-Item -ItemType Directory -Force -Path $destination | Out-Null
Copy-Item "target\release\tabularis-mongodb-plugin.exe" $destination
Copy-Item "manifest.json" $destination
```

Restart Tabularis after installation. The plugin appears as **MongoDB Atlas** and can coexist with the official MongoDB plugin.

## Build and test

Prerequisites:

- Rust toolchain with Cargo
- network/DNS access when testing a live `mongodb+srv://` URI
- an Atlas IP access-list entry for the client machine when testing Atlas

Run the quality gates:

```bash
cargo fmt
cargo test
cargo build --release
```

The binary is written to `target/release/tabularis-mongodb-plugin` (or `.exe` on Windows).

## Manual JSON-RPC test

Build first, replace the placeholder credentials and cluster locally, then run:

```bash
echo '{"jsonrpc":"2.0","method":"test_connection","params":{"params":{"connection_string":"mongodb+srv://user:pass@cluster0.xxxxx.mongodb.net/mydb?retryWrites=true&w=majority&authSource=admin","database":"mydb"}},"id":1}' | ./target/release/tabularis-mongodb-plugin
```

Expected response:

```json
{"jsonrpc":"2.0","result":{"success":true},"id":1}
```

A real request succeeds only when DNS, the Atlas IP access list, credentials, database permissions, and TLS trust are configured correctly.

## Query syntax

The `execute_query` method accepts MongoDB shell syntax:

```javascript
db.users.find({"age": {"$gt": 18}})
db.orders.findOne({"status": "pending"})
db.orders.aggregate([
  {"$match": {"status": "shipped"}},
  {"$group": {"_id": "$customer_id", "total": {"$sum": "$amount"}}}
])
db.users.countDocuments({"role": "admin"})
```

For collection browsing, Tabularis can also send `SELECT * FROM collection_name`; the plugin converts it to `find({})`.

## Supported operations

The fork preserves the original capabilities: `test_connection`, `get_databases`, `get_schemas`, `get_tables`, `get_columns`, `get_foreign_keys`, `get_indexes`, `execute_query`, `insert_record`, `update_record`, `delete_record`, `get_schema_snapshot`, batch schema methods, MongoDB DDL-equivalent generation, and `drop_index`.

## Security guidance

For financial or other sensitive environments:

- Never commit, paste into issues, or publish connection strings containing credentials.
- Store the Atlas URI in an approved secret manager or the protected Tabularis connection store.
- Use an Atlas IP access list with the narrowest possible CIDR; do not use `0.0.0.0/0` in production.
- Create a dedicated database user with least-privilege roles for only the required databases and operations.
- Use TLS and verify the operating system trust store; never disable certificate validation.
- Prefer SCRAM-SHA-256 or the organization-approved Atlas authentication mechanism.
- Use separate users and clusters for development, staging, and production.
- Enable Atlas auditing, alerts, and connection monitoring where available.
- Rotate passwords and revoke exposed users immediately if a URI is disclosed.
- Review CRUD permissions carefully before using Tabularis against production financial data.

The plugin never intentionally logs a connection URI. URI-construction errors do not echo the supplied URI, and MongoDB URIs found in driver errors have their user information masked before being returned through JSON-RPC.
